# Чеклист smoke-проверки после миграции калькуляторов на backend

Проверять после каждого деплоя или изменения proxy/hook.

## Общее

- [ ] В `.env` заданы `CALC_API_BASE_URL` и `CALC_API_KEY` (или в dev без них показывается 503 и сообщение об ошибке).
- [ ] Страницы калькуляторов открываются без белого экрана (маршруты и layout не сломаны).

## Мигрированные калькуляторы (backend)

Для каждого пункта: открыть страницу, изменить 1–2 поля, убедиться, что результат обновляется и нет ошибок в консоли.

- [ ] **osago** — `/osago`: параметры ТС, регион, водители; итог и диапазон стоимости отображаются; при ошибке API показывается сообщение.
- [ ] **auto-loan** — `/auto-loan`: сумма, взнос, ставка, срок; ежемесячный платёж и график; при загрузке — «Загрузка…».
- [ ] **mortgage** — `/mortgage`: стоимость, взнос, ставка, срок; платёж и переплата; при загрузке — «Загрузка…».

## Калькуляторы gap (локальный расчёт)

Убедиться, что работают как раньше (без вызова backend):

- [ ] **vat** — расчёт НДС.
- [ ] **gk395** — неустойка по 395.
- [ ] **penalty-ddu** — пеня по ДДУ.
- [ ] **subsistence-minimum** — прожиточный минимум.
- [ ] **alimony-indexation** — индексация алиментов.
- [ ] **inflation** — инфляция.

## Регрессии

- [ ] Нет дублирования запросов при быстром изменении полей (отмена предыдущего запроса).
- [ ] Сообщения об ошибках не содержат чувствительных данных (ключей, токенов).
- [ ] SEO: заголовки и мета не пропали на страницах калькуляторов.
